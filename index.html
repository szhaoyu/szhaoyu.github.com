  • 19 December - MR vs SQL

    Map-Reduce framework or algrithm is good for indexing scraped web pages or proccessing server logs.

    It’s proccess requirement is simple, but data scale is large.

    But for complex structured data proccessing, hadoop’s map-reduce is little inefficent.

    Hadoop’s working context is Input file and out file. RDBMS’s context is more than that, it used more memory to compute.

    Here compare the function of Map-Reduce to SQL

    Map function equals to SQL’s ‘where’ predictor.

    Reduce function equals to SQL’s ‘group by’ operator.

    And SQL has much more function semantics than hadoop’s single Map-Reduce algrithm.

  • 19 December - Connection between dockers accross different hosts

    Dockers in same host can ping each other by IP, because they all use the same network bridge.

    How do dockers located on different hosts connect each other?

    1. bridge on same network field

    Host1 ： 10.211.55.3 eth0

    Host2 ： 10.211.55.5 eth1

    apt-get install bridge-utils

    Host1: $ sudo brctl addbr br0 Host2: $ sudo brctl addbr br0

    Host1: $ sudo ifconfig br0 10.211.55.10 netmask 255.255.255.0 Host2: $ sudo ifconfig br0 10.211.55.20 netmask 255.255.255.0

    Host1: $ sudo brctl addif br0 eth0 Host2: $ sudo brctl addif br0 eth0

    $sudo vim /etc/default/docker

    Host1: DOCKER_OPTS=” -b=br0 –fixed-cidr=‘10.211.55.64/26‘ “ Host2: DOCKER_OPTS=” -b=br1 –fixed-cidr=‘10.211.55.128/26‘ “

    2. Open vSwitch requirements: two network-cards, Host-Only & NAT

    3. weave requirements: two network-cards, Host-Only & NAT
